a. What is GIT? Why do we need source code Version Control System?

Git is a distributed version control system. Projects which use multiple developers at the same need GIT.
By using GIT, it can be ensured that there are no code conflicts between the developers.

b.Describe following terms: 1) local repo 2) remote repo 3) how to use GIT in Team Project – multiple developers work on the same project at the same time

1) local repo
It is on your computer and has all the files and their commit history, enabling full diffs, history review, and committing when offline.

2) remote repo
It is centralized server.The remote repo is usually not on your machine and is the one shared by the team.

3) how to use GIT in Team Project – multiple developers work on the same project at the same time
Developers work on each local repo. The team “pushes” commits to remote repo when ready to share with the team.
While one of your remote repos could be another team member’s local repo, in a corporate environment at least one is typically a Git repo on a server anointed as the central/official repo.
